## Context

plantuml-js is a TypeScript port of PlantUML. Stack: TypeScript, Node.js
ESM, vitest, vite. Scripts run via `jiti` (already a devDependency).

Existing `package.json` scripts:
```json
"visual:capture": "jiti tests/visual/capture-reference.ts",
"visual:compare": "playwright test --config tests/visual/playwright-visual.config.ts"
```

These cover only 7 canonical examples. We are adding a corpus-scale
pipeline. See `planning/visual-qa.md` for the full architecture.

## Task

```json
"visual":          "npx serve tests/visual"
```

`visual:capture` replaces the existing entry (old script fetched 7
canonical files; new script processes full manifests).
`visual:compare` stays unchanged.

### 2. Update `planning/visual-qa.md`

Add a section "## Adding a new diagram type" after the existing content:

```markdown
## Adding a new diagram type

When a new diagram type is implemented in `src/`:

1. Add its `@start` keyword (or body marker) to the classifier in
   `scripts/classify-corpus.ts` (add a case to the `detectType`
   function).
2. Re-run `npm run visual:classify` to regenerate manifests.
3. Run `npm run visual:capture` to fetch reference PNGs for the new
   type (skips already-captured files).
4. Run `npm run visual:build` to regenerate HTML pages.
5. Update the status column in `planning/phases.md` and the coverage
   table in `planning/visual-qa.md`.

No changes needed to the HTML templates or page generator — new types
are picked up automatically from manifests.
```

Also update the **Status** column in the coverage table to mark the
7 implemented types as ✅:
sequence, class, component, state, usecase, activity, object → ✅
