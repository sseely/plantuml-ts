## D2 — Composite states port GroupMakerState's two-path mechanism

Verified against oracle dumps (bemena-23-zebu249): an "autonom"
composite (no links crossing its boundary — `svek/GroupMakerState.java`,
`InnerStateAutonom.java`) is laid out in a CHILD svek pass, dumped
BEFORE the parent graph, and re-enters the parent as a fixed-size
`shape=rect,style=rounded` node carrying the child layout's total
dims. Non-autonom composites become nested cluster envelopes
(`cluster{N}a` wrapper → `p0`/`p1` concurrent-region parts → `{N}` →
`i` inner, plus `zaent` point anchors — `ClusterDotString.java`). Our
engine must run child passes in the oracle's dump order so the
comparator pairs graph #i with svek-(i+1). T1 documents the exact
rules in mechanisms.md before T3/T4 implement.
